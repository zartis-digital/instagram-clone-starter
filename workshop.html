Getting Started

Prerequisites

  • Node.js 20+
  • bun installed globally
  • Git

Tech Stack

React 19 Vite TanStack Router TanStack Query Tailwind CSS v4 shadcn/ui

Setup

Terminal
git clone https://github.com/zartis-digital/instagram-clone-starter cd instagram-clone bun install

Environment Variables

Create a .env file in the project root to point the frontend at the production API:

.env
VITE_API_URL=https://fqg27fj0idic0tmq91ssdh20.178.105.11.199.sslip.io VITE_WS_URL=wss://fqg27fj0idic0tmq91ssdh20.178.105.11.199.sslip.io VITE_STORAGE_PUBLIC_URL=https://minio-api-kqf7qd.178.105.11.199.sslip.io/instagram-uploads

Then start the dev server:

Terminal
bun dev

App runs at http://localhost:3000. Launch Claude Code with claude in the project root.

Project Structure

src/ routes/ ← File-based routing (TanStack Router) components/ ← React components queries/ ← TanStack Query hooks lib/ ← API client, utilities hooks/ ← Custom React hooks

Key Files

Routes are in src/routes/, queries in src/queries/, and the API client helper is at src/lib/api.ts.

Important: The backend API is already deployed. You only work on the frontend. API docs are available at https://fqg27fj0idic0tmq91ssdh20.178.105.11.199.sslip.io/swagger.

Plan Mode — Think Before You Code
Use Plan Mode to design your approach before writing code
~10 min
  1. Enter Plan Mode
    Claude Code
    /plan

    Plan Mode tells Claude to research and design an approach WITHOUT making any changes. It explores the codebase, reads relevant files, and proposes a step-by-step implementation plan. You review the plan and only then approve it for execution.

    You can also toggle Plan Mode with Shift+Tab. Use it whenever a feature touches 3+ files or involves complex logic.
  2. Plan your first feature — Let's plan the Create Post feature:
    Prompt (in plan mode)
    I need to implement the Create Post feature. The create post dialog exists but the upload logic, image preview, caption input, and submit handler are missing (marked with TODOs). Plan how to implement this. Check the API swagger docs for the POST /posts endpoint and the upload flow.
  3. Review the plan — Claude will propose:

    Which files to modify, what components/hooks to create, the data flow (upload → get URL → create post), and edge cases to handle. Ask questions, request changes, or approve when satisfied.

  4. Exit Plan Mode — When you're happy with the plan:
    Claude Code
    /plan
    Toggle /plan again to exit. Now Claude will execute the plan and make the actual code changes.
Stuck? Click for help
Plan Mode is especially valuable when you're unfamiliar with a codebase. It lets Claude do the research for you and propose an approach you can review before any code is written.

Part 2: Building the Orchestration Layer

Build a team of specialized AI agents that collaborate to deliver software. A product agent defines requirements, skills persist and flesh out issues, and developer and QA agents implement and verify. (~30 min)

5
Product Agent
A product manager agent that defines requirements and creates issues
~10 min

Agents are specialized personas that Claude can adopt. Each agent is a markdown file in .claude/agents/ with a system prompt that defines its role, expertise, and behavior. Claude automatically discovers them — you can ask Claude to use one or it may choose the right agent on its own.

The product manager agent is responsible for turning vague feature ideas into structured, actionable issues that the developer agent can pick up and implement.

Prompt
Create a product manager agent at .claude/agents/product-manager.md. It should: - Act as a senior product manager with strong domain knowledge - Write user stories in the format: "As a [user], I want [goal] so that [benefit]" - Break features into small, focused issues and save them to docs/issues/ using the /create-issue skill - Think about edge cases, error states, and out-of-scope items - Reference the Swagger API docs to understand what's possible - Never write code — only define requirements and specs
Claude automatically discovers agents in .claude/agents/. You can ask Claude to "use the product manager agent" or it may select it automatically based on your request.
  • Created .claude/agents/product-manager.md
  • Understand what the agent produces (user stories, specs, issues)

Create Issue Skill
A /create-issue slash command that saves issues to docs/issues/
~5 min

Skills are custom slash skills defined in .claude/skills/ that automate repeatable workflows. The /create-issue skill gives the product manager (and any other agent) a standard way to persist requirements as markdown files that the developer agent can later pick up.

Prompt
Create a skill at .claude/skills/create-issue.md with this frontmatter and content: --- description: Create a new issue markdown file in docs/issues/. Use when a feature, bug, or task needs to be tracked as a work item. argument-hint: [title] [description] allowed-tools: Write Bash(mkdir *) disable-model-invocation: true --- When invoked with /create-issue: 1. Accept a title and short description as input 2. Generate a URL-friendly slug from the title (lowercase, hyphens, no special chars) 3. Create a markdown file at docs/issues/<slug>.md with these fields: title, description, acceptance criteria, priority (low/medium/high), status: open 4. Create the docs/issues/ directory if it doesn't exist 5. Print the path of the created file
Skills run as slash skills in the current conversation context. Any agent can invoke /create-issue directly, making it a shared contract between the product manager, developer, and QA agents.
  • Created .claude/skills/create-issue.md
  • Tested /create-issue manually and saw the file appear in docs/issues/

A /interview-requirements skill that fleshes out feature details before filing an issue
~10 min

Writing good requirements is a dialogue, not a monologue. This skill combines the product manager agent's expertise with interactive questioning to ensure every issue is well-defined before any code is written.

Prompt

Adopt the product manager agent persona 2. Ask the user 3-5 targeted questions using the AskUserTool to clarify: scope, target users, edge cases, acceptance criteria, and priority 3. This skill combines a specialized agent persona with interactive questioning — a powerful pattern for ensuring requirements are clear before any code is written. The AskUserTool lets Claude present structured choices to gather precise answers.

Agent vs Skill — when to use which?
Agents change Claude's persona for the conversation — use them when you want Claude to think and behave differently (e.g., as a QA tester vs a developer). Skills are one-shot automations that run a specific workflow — use them for repeatable tasks like gathering requirements, creating issues, or committing.

Optimistic updates tip
Optimistic updates make the UI feel instant. The pattern is: update the query cache immediately on click, then let the mutation run in the background. If it fails, the cache reverts. TanStack Query makes this easy with onMutate, onError, and onSettled callbacks.

Part 5: Tips, Tricks & Resources

Useful patterns, keyboard shortcuts, and links to keep learning.

Tips & Tricks

Prompting

  • Be specific about files — "In src/components/feed/post-actions.tsx, add a like button" works better than "add a like button somewhere".
  • Reference the Swagger docs — Telling Claude to "check the Swagger docs for the endpoint" gives it the exact request/response shapes instead of guessing.
  • Iterate, don't restart — If Claude gets something wrong, tell it what's wrong in the same conversation. It has full context of what it just did.
  • Paste error messages — Copy-paste the full error. Claude is great at diagnosing stack traces and TypeScript errors.
  • Use screenshots — "Take a screenshot and tell me what's wrong" is a powerful debugging prompt when you have Playwright MCP configured.

Workflow

  • Plan Mode for complex features — Use Shift+Tab to toggle Plan Mode before starting a multi-file feature. Review the plan, then toggle back to execute.
  • One feature per conversation — Start a new conversation (/clear) between features to keep context clean and focused.
  • Compact long conversations — If a conversation gets long, use /compact to summarize the history and free up context space.
  • Let the agent cook — For big features, give Claude the full requirements and let it work. Review the result rather than micromanaging each step.
  • Use agents for different hats — Switch to the QA agent after implementing a feature to catch bugs you might have missed.

Keyboard shortcuts

  • Shift+Tab — Toggle Plan Mode
  • Esc — Cancel current generation
  • Tab — Accept Claude's suggestion
  • Ctrl+C — Exit Claude Code

Useful links

  • API Swagger Docs — Interactive API documentation with all available endpoints and schemas
  • Claude Code Documentation — Official docs covering installation, configuration, CLAUDE.md, MCP servers, agents, and more
  • Anthropic Skilljar Courses — Free courses on prompt engineering, Claude best practices, and building with Claude
